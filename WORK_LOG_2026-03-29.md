# 工作日志 - 2026-03-29

## 项目：多人实时同步记账网站

---

## 1. 项目概述

为用户创建了一个支持多人实时协作的在线记账工具，适合朋友聚会、旅行等场景共同记账。

### 核心功能
- 房间系统：生成6位房间码，邀请朋友加入
- 实时同步：基于 Socket.io，多人同时编辑即时更新
- AA自动计算：自动计算每人应收/应付，生成转账建议
- 数据持久化：SQLite存储，永久保存

---

## 2. 技术栈

- **后端**: Node.js + Express + Socket.io
- **数据库**: SQLite3
- **前端**: 原生 HTML + CSS + JavaScript
- **部署**: Railway（免费，含持久化存储）

## 4. 开发过程

### 4.1 初始版本（本地版）
- 先创建了单机版记账工具（expense-tracker/）
- 使用 localStorage 存储，无法多人同步

### 4.2 重构为在线版
- 添加 Express 后端
- 使用 Socket.io 实现实时通信
- 使用 SQLite 持久化存储
- 实现房间系统（6位随机码）

### 4.3 部署配置

#### Render（放弃）
- 配置 render.yaml
- 发现 Free 计划无持久化存储，数据会随睡眠清空

#### Railway（采用）
- 配置 railway.json
- 自动创建 Volume（挂载路径 /data）
- 数据库路径改为 `/data/expenses.db`
- 数据持久保存，重启不丢失

---

## 5. 遇到的问题及解决方案

| 问题 | 解决方案 |
|------|----------|
| npm 缓存权限错误 | 使用临时缓存目录 `npm install --cache /tmp/npm-cache` |
| SSH 推送失败（Host key 变更） | 更新 known_hosts：`ssh-keyscan github.com >> ~/.ssh/known_hosts` |
| Render Free 无持久化存储 | 改用 Railway，Free 计划支持 Volume |
